BirbShare Admin Guide

Admin Dashboard Overview

Upon logging into the BirbShare Admin Panel, you are greeted by the Main Dashboard. This page acts as your "Mission Control," providing a high-level snapshot of your platform's health, revenue, and user activity.

Quick-View Metrics (Top Row)

The top section displays real-time counters. Use these cards to gauge system load and business performance at a glance.

  • Transfers & Files: Tracks the volume of content being shared. A sudden spike here may indicate a viral file or increased server load.
  • Total Size: Monitors the aggregate storage space used across the server (e.g., 781.1 MB).

    Note: Keep an eye on this metric to anticipate when you need to scale your storage infrastructure.

  • Pageviews & Downloads: Measures engagement. High pageviews with low downloads may indicate issues with the download page UI.
  • Users: The total count of registered accounts for the current month.
  • Earned (Revenue): Displays the gross revenue (e.g., 6.19 USD) processed in the current period.

Latest Users & Online Status

This table provides immediate visibility into user growth and current platform usage.

  • User Identity: Displays the user's avatar and email address.
  • Status Indicator:
    • Active (Green): The user has verified their email and is in good standing.
    • Inactive/Banned (Red/Gray): The user requires attention or has been restricted.
  • Plan Details: Quickly identify if the new signup is on a Free or Premium tier.
  • Metadata Icons: The icons under "Details" represent the user's environment (Browser, OS, Country, IP) to help with debugging or fraud detection.

Admin Notifications

The Admin Notifications feed acts as a real-time log of critical system events.

  • New Signups: alerts you immediately when a new user registers.
  • System Alerts: (If configured) This section will also display critical errors or payment failures.
  • Action: Click on a specific notification to jump directly to the relevant user profile or log entry.

Latest Payments

For platforms monetizing via subscriptions, this section tracks incoming cash flow.

  • Transaction Details: Shows the User, the Plan purchased (e.g., Premium), and the exact amount.
  • Type & Gateway: Indicates if the charge was a One-time payment or Monthly subscription, and which processor handled it (e.g., Stripe, PayPal).
  • Quick Actions:
    • Use the Trash icon to manually void or hide a transaction record (use with caution).
    • Use the Options menu (â‹®) to view the full invoice or issue a refund.

Navigation Sidebar Reference

The sidebar provides quick access to every aspect of the BirbShare platform. The menu is arranged from general administration at the top to granular technical settings at the bottom.

General Administration

  • Admin dashboard: The landing page. Displays the "Mission Control" view with high-level metrics, active users, and recent revenue snapshots.
  • Users: The master list of all registered accounts. Use this to search for specific members, edit profile details, assign roles (e.g., Moderator), or ban abusive users.
  • Settings: The global configuration hub. Manage core site attributes including the site name, logo, SEO metadata, and default upload limits.
  • Plans: Configure your monetization tiers. Create, edit, or delete subscription packages (e.g., Free vs. Premium) and define the storage quotas or expiration rules for each.
  • Languages: Localization management. Add new languages to your site or translate specific UI strings to support a global user base.

Communication & Extensions

  • Broadcasts: A mass-communication tool. Use this to send platform-wide announcements or marketing emails to all registered users.
  • Users notifications: Manage the specific alerts users receive within their dashboard (e.g., "Your file has expired" or "Payment successful").
  • Plugins: The extension manager. Install, activate, or configure third-party add-ons to expand site functionality without altering core code.
  • Statistics: Detailed analytics. Access in-depth charts regarding server storage usage, visitor geography, and traffic trends over time.

Content Management (CMS)

  • Resources: (Dropdown) Manage the Help Center or Knowledge Base articles to assist users with common questions.
  • Blog: (Dropdown) A built-in CMS to publish news, updates, and articles to your public-facing blog pages.
  • API docs: Auto-generated documentation for the BirbShare API. Useful for developers or users looking to integrate your service with external tools.

Finance & Monetization

  • Codes: Manage promotional strategies. Create discount coupons for paid plans or generate invite codes for private registration.
  • Taxes: Configure regional tax rules (such as VAT or GST) to ensure compliance for payments processed in specific countries.
  • Payments: The financial ledger. View a chronological history of all incoming transactions, download invoices, or process refunds.

File & Project Management

  • Transfers: A log of all transfer sessions initiated by users. Use this to moderate activity or investigate reports of prohibited content.
  • Files: A direct file-system view. Allows admins to browse hosted assets and permanently delete specific files from the server.
  • Projects: Manage user workspaces or grouped collections of files.

Technical Configuration

  • Pixels: Marketing integration. Add tracking IDs for services like Google Analytics or Facebook Pixel to track conversion rates.
  • Domains: Manage custom branding. Configure settings for users who wish to connect their own custom domains to their BirbShare profiles.
  • Notif. handlers: Configure the delivery drivers for system emails (e.g., SMTP settings, SendGrid, Mailgun) to ensure emails reach user inboxes.
  • Users logs: Security auditing. View a raw log of user actions, including login times and IP addresses, to detect suspicious activity.

Settings - Main

The Settings > Main page allows you to configure the core identity of your site, including branding assets, default user preferences, and system-wide visual themes.

Site Identity & Branding

These fields control how your site appears to users and how it looks when shared on social media.

  • Website Title: The global name of your application (e.g., BirbShare). This appears in the browser tab and is used in system emails.
  • Logo for light theme: Upload the version of your logo optimized for light backgrounds.
  • Logo for dark theme: Upload the version of your logo optimized for dark backgrounds (usually with lighter text/icons).
  • Logo for sent emails: A specific logo file used in transactional emails (Welcome emails, password resets, etc.). This ensures your branding looks correct in email clients.
  • Favicon: The small icon displayed in the browser tab next to the site title.
  • Opengraph Image: The "preview card" image that appears automatically when someone shares a link to your site on platforms like Facebook, Twitter/X, or Discord.

    Format Note: Most image fields accept .jpg, .jpeg, .png, .svg, .gif, .webp, and .avif. The maximum file size is 512 MB.

Regional & System Defaults

These settings determine the default experience for new visitors and unauthenticated users.

  • Default timezone: Sets the server baseline time (e.g., UTC). This is the default for new users until they customize it in their own profile settings.
  • Default theme style: Choose whether the site loads in Light or Dark mode by default for new visitors.
  • Default language: Sets the primary language for the site interface (e.g., english - en).
  • Default results per page: Controls the density of data tables across the admin and user panels (e.g., 100 items per page).
  • Default results order: Determines if lists (like files or users) are sorted by Ascending (Oldest first) or Descending (Newest first) by default.

App-wide Settings

The App-wide Settings section controls the foundational behavior, navigation logic, and global feature availability of your BirbAI installation. These toggles allow you to enable or disable core system functionalities without needing to touch code.

  • Enable spotlight system for admins Enables the global command palette for administrators. When active, admins can press CMD + K (Mac) or CTRL + K (Windows) to open a search modal and quickly navigate to different pages.
  • Enable spotlight system for users Enables the global command palette for regular users. This allows them to quickly search for features or navigate their account using CMD/CTRL + K.
  • Enable white labeling system Unlocks the ability for specific users (usually on high-tier plans) to customize the account branding. This allows them to replace the default logo, footer, and page titles with their own brand identity.
  • Enable theme style selector Adds a toggle button in the website footer, allowing users to manually switch between Light and Dark visual themes.
  • Enable automatic language detection When a visitor arrives, the system will detect their browser's preferred language. If that language is supported by your site, they will be automatically redirected to the localized version.
  • Enable users API system Allows users to generate and manage their own API keys. Enable this if you want to let developers build third-party applications or integrations using your platform.
  • Track email views & clicks for Broadcasts Adds tracking pixels to system emails to measure open rates and link clicks.

    Note: Enabling this increases the load on your server and requires more database storage.

  • Enable breadcrumbs Displays a navigation trail at the top of pages (e.g., Home > Dashboard > Settings). This helps users understand where they are within the application structure.
  • Display pagination regardless of page count
    • Enabled: The pagination controls are always visible, even if there is only one page of results.
    • Disabled: Pagination controls are hidden unless there are at least two pages of content.
  • Allow search engines indexing Controls whether public pages of your site are visible to search engines like Google or Bing.
    • On: Your robots.txt file will allow indexing.
    • Off: Search engines will be instructed to ignore your site (useful for private or development environments).
  • Allow AI scraping Controls whether AI data bots (like OpenAI, Google Gemini, or Bing) are allowed to scrape your public content for model training.
  • Force HTTPS Forces all site traffic to use the secure https:// protocol.

    Warning: Only enable this after you have confirmed that the SITE_URL value in your config.php file starts with https://. Enabling this prematurely can lock you out of the site.

  • Iframe embedding allowed domains Controls security settings for embedding your site elsewhere.
    • all: Allows your site to be embedded in an iframe on any domain.
    • none: Completely disables iframe embedding (recommended for security).
    • Specific URLs: Enter comma-separated domains to allow embedding only on trusted sites.
  • Title separator Defines the symbol used in the browser tab title to separate the page name from the website name (e.g., enter - for titles like "Dashboard - BirbShare").

Index Settings

This page allows you to customize the content blocks displayed on your default landing page and optionally redirect the root domain to a completely custom external site.

Content Toggles

Use the toggles below to control which informational blocks are visible to visitors on the index (home) page.

  • Display plans on the index page
    • On: Shows the pricing table and plan features section on the landing page.
    • Off: Hides the pricing details from the homepage.
  • Display testimonials on the index page
    • On: Displays a section featuring user reviews and social proof.
    • Off: Hides the testimonials block.
  • Display FAQ on the index page
    • On: Shows the Frequently Asked Questions section.
    • Off: Hides the FAQ block.
  • Display latest blog posts on the index page
    • On: Displays a widget showing a preview of recent articles from your Blog.
    • Off: Hides the blog preview widget.

Custom Index Page Configuration

  • Custom index page URL This field allows you to completely bypass the default BirbAI landing page and redirect all root traffic to an external URL (e.g., a page hosted on WordPress or a separate marketing site).
    • To Enable: Enter the full URL of the custom landing page (e.g., https://example.com/). This will disable the original landing page and redirect all incoming requests.
    • To Disable: Leave the field empty. The system will then revert to displaying the original BirbAI landing page and the content blocks enabled above.

Maintenance Settings

Use these settings to enable the application-wide Maintenance Mode and customize the message presented to users while the site is down for updates, bug fixes, or new feature rollouts.

Maintenance Mode Activation

  1. Enable maintenance mode
    • Function: This is the master switch for taking the site offline.
    • Behavior: When enabled, all users (non-Admins) attempting to access any page will be redirected to the custom maintenance screen.
    • Exceptions: Access is preserved for essential administrative and recovery functions: the Login page, Lost Password page, and Reset Password pages remain accessible. Admins can log in and view the site normally.

Maintenance Page Customization

Customize the content and action link displayed on the maintenance screen.

  1. Maintenance title The main heading displayed prominently on the maintenance page (e.g., BirbShare).
  2. Maintenance description The body text that informs your users about the reason for the downtime (e.g., "BirbShare is currently undergoing maintenance. We are updating the software and integrating the latest APIs.").
  3. Action button text The text displayed on the call-to-action button (e.g., Visit Birb.com).
  4. Action button URL The destination link for the action button. This is typically used to direct users to a status page, a marketing site, or a social media channel for updates (e.g., https://birb.com).

Other Settings

This section manages miscellaneous core system variables, including legal obligations, system performance caching, and the configuration for administrative AI services.

Other Settings Reference

Custom 404 page URL

  • Function: Disables the default "Page Not Found" error page.
  • Usage: Enter a full URL (e.g., https://example.com/) to redirect users hitting a broken link to a custom error page. Leave the field empty to revert to the system's default 404 page.

Terms and Conditions URL

  • Function: Provides the legally required link to your Terms and Conditions.
  • Display: This link is displayed to the user during the sign-up process and when they are making a payment.

Privacy Policy URL

  • Function: Provides the legally required link to your Privacy Policy.
  • Display: This link is displayed to the user during the sign-up process and when they are making a payment.

Preview chart cache

  • Sets the duration (in hours) for which analytical chart data is cached. Lowering this value provides more real-time data but increases server load.

Preview chart time frame

  • Defines the maximum time period (in days) that the dashboard and statistics charts cover. Data older than this limit will not be included in the default chart visualization.

Account avatar size limit

  • Sets the maximum file size (in MB) allowed for user avatar uploads.

  • Note: This limit cannot be set higher than the maximum upload size configured on your actual server.

OpenAI API key

  • Enter a valid OpenAI API key here.
  • Usage: This key is specifically used for the automatic translation functionality found within the admin panel language packs and system assets.

OpenAI model

  • Select the specific OpenAI model to be used for the internal admin translation feature (e.g., gpt-5).

Sitemap

  • Sitemap URL: This read-only field displays the link to your generated sitemap (e.g., https://birbshare.com/sitemap). Submit this URL to search engines (like Google Search Console) to improve your SEO indexing.

Saving Your Changes

After making any modifications, you must click the blue Update button at the very bottom of the page to apply them.

Settings - Users

This section manages all user-related system settings, including new user registration, login and social login behavior, email confirmation, cookie timeframes, and provides access to Cleanup and Anti spam measures sub-sections.

Enable new users registration

  • Function: Controls user access to the platform.
  • Behavior: Allows people to register on your platform. Disabling this toggle completely removes the registration page, making the platform invite-only or admin-only.

Exclusive login/registration with Social logins

  • Function: Restricts the available login methods.
  • Behavior: When enabled, users can only register or log in using third-party social providers (e.g., Google, Facebook). Standard email/password registration is disabled.

Social logins require password

  • Function: Enhances account security for social sign-ups.
  • Behavior: When enabled, users who first register via a social login will be required to set an account password.

Display newsletter sign up checkbox on registration page

  • Function: Adds a marketing opt-in during account creation.
  • Behavior: Displays a newsletter sign-up checkbox on the primary registration page.

Display newsletter sign up checkbox on account settings page

  • Function: Provides a persistent opt-in/opt-out option.
  • Behavior: Displays a newsletter sign-up checkbox within the user's personal account settings page.

Auto-check "Remember me" on login page

  • Function: Controls the default state of the login session persistence checkbox.
  • Behavior: When enabled, the "Remember me" checkbox on the login page will be checked by default.

"Remember me" cookie time frame

  • Function: Sets the duration for session persistence.
  • Behavior: Defines how long (in days) the user's login session remains active when the "Remember me" box is checked. Default is 90 days.

Enable usage of email aliases

  • Function: Allows for flexible email management.
  • Behavior: Permits people to register or use email aliases (e.g., hello+admin@example.com or user+alias@email.com).

Email confirmation

  • Function: Controls account security and verification upon sign-up.
  • Behavior: When enabled, an email confirmation link is sent when a user registers or changes their email address. The account remains unconfirmed until the link is clicked.

Welcome email

  • Function: Controls introductory communication.
  • Behavior: Sends out a welcome email automatically when a new user successfully completes registration.

Cleanup Settings

This page manages automated maintenance routines and user deletion policies based on inactivity and confirmation status, helping you keep your user database lean and secure.

Auto delete unconfirmed users

  • Function: Automatically deletes user accounts that failed to confirm their email address.
  • Duration: Enter the amount of days an unconfirmed account will exist before being automatically deleted.
  • Note: Since these users are unconfirmed, no deletion reminders will be sent to them. Set the value to 0 to permanently disable this feature.

Auto delete inactive users

  • Function: Automatically deletes user accounts that have been inactive for a specified period.
  • Duration: Enter the amount of days a user can be inactive before their account is deleted.
  • Restriction: This feature is only valid for Free users. Paid subscribers are not subject to auto-deletion for inactivity. Set the value to 0 to disable this feature.

User deletion reminder

  • Function: Sends a warning email to users scheduled for deletion due to inactivity.
  • Duration: Enter the number of days prior to the deletion date that the user should receive the reminder email.
  • Note: Set the value to 0 to disable this reminder system.

Anti spam measures

This page allows administrators to configure various security and throttling mechanisms to prevent automated bot activity,fraudulent signups, and brute-force attacks.

Blacklisted domains

  • Function: Prevents registration and login attempts from specific email domains.
  • Usage: Enter domain names (e.g., spamdomain.com, tempmail.co) separated by commas. Users will be unable to register or log in if their email address is associated with any domain in this list.

Blacklisted IPs

  • Function: Blocks access based on the user's IP address.
  • Usage: Enter IP addresses separated by commas. Users will be unable to register or log in if their IP address is included in this list.

Select blacklisted countries

  • Function: Restricts access based on geographical location.
  • Usage: Select countries from the list that you wish to block. Users will be unable to register or log in from any of the selected countries. Detection is based on the user's IP address.

Login lockout is enabled

  • Function: Activates brute-force protection for the login page.
  • Behavior: When enabled, login attempts are throttled after repeated failures.

Max login retries

  • Function: Defines the number of failed login attempts allowed before the lockout takes effect.
  • Default: 5 retries.

Lockout time

  • Function: Defines the duration the user's IP/account is blocked after reaching the max retries limit.
  • Unit: Minutes.

Password reset lockout is enabled

  • Function: Activates throttling protection for the password reset functionality.
  • Behavior: Limits the number of times a user can attempt to reset their password within a given timeframe.

Max reset retries

  • Function: Defines the number of failed password reset attempts allowed before the lockout takes effect.
  • Default: 5 retries.

Lockout time

  • Function: Defines the duration the user's IP/account is blocked after reaching the max reset retries limit.
  • Unit: Minutes.

Resend activation lockout is enabled

  • Function: Activates throttling protection for resending account activation emails.
  • Behavior: Limits the number of times a user can request a new activation email.

Max resend retries

  • Function: Defines the number of resend requests allowed before the lockout takes effect.
  • Default: 5 retries.

Lockout time

  • Function: Defines the duration the user's IP/account is blocked after reaching the max resend retries limit.
  • Unit: Minutes.

Register lockout is enabled

  • Function: Activates throttling protection to prevent spam registrations.
  • Behavior: Limits the number of registrations allowed from a single IP address or network within a set period.

Max registrations

  • Function: Defines the maximum number of successful registrations allowed before the lockout takes effect.
  • Default: 5 registrations.

Lockout time

  • Function: Defines the duration the user's IP is blocked from registering after reaching the max registrations limit.
  • Unit: Days.

Enable blog system

Enable blog share post buttons

  • Behavior: Allows users to easily share articles on various social media platforms.

Enable blog search widget

  • Function: Adds a dedicated search bar widget to the blog sidebar.
  • Behavior: Allows visitors to search for specific content within the blog.

Enable blog categories widget

  • Function: Adds a navigation widget listing post categories to the blog sidebar.
  • Behavior: Helps users filter posts by topic.

Enable blog popular posts widget

  • Function: Adds a widget listing the most-viewed articles.

Enable blog views count to be displayed

  • Function: Controls the visibility of post view analytics.

Enable blog ratings system

Blog system columns

  • Function: Defines the layout structure of the blog post listing page.

Resources

This section controls the configuration and visibility of the Resources or static Pages system (often used for Help/About pages).

Enable resources (pages) system

  • Function: The master switch for the static Pages feature.
  • Behavior: When enabled, the Resources section becomes functional and manageable via the Admin Sidebar.

Enable pages share post buttons

  • Function: Adds social sharing links to static resources/pages.
  • Behavior: Allows visitors to share help articles or static pages on social media.

Enable pages popular posts widget

  • Function: Adds a widget to the resources sidebar listing the most frequently viewed pages.

Enable pages views count to be displayed

Settings - Transfers

The Settings > Transfers page controls the core behavior of file sharing on BirbShare. Here, you can configure how links are generated, set upload limitations, and toggle specific features like custom domains or email transfers.

URL Generation & Branding

Control the appearance of the public download pages.

  • Random URL length: Defines the number of characters in the automatically generated link slug (e.g., a value of 7 results in birbshare.com/aBcDeFg).
    • Recommendation: Keep this between 6â€“8 characters to balance short URLs with enough unique combinations to avoid collisions.
  • Branding: This text (or HTML) appears in the footer of the transfer download page.
    • Dynamic Variables: You can use specific placeholders that automatically populate based on context:
      • {{WEBSITE_TITLE}}: Inserts your site name.
      • {{URL}}: Inserts the link to the file.
      • {{AFFILIATE_URL_TAG}}: Inserts the user's affiliate ID if applicable.

Upload Behavior & Performance

These settings affect the server performance and the user upload experience.

  • Enable email transfers: If enabled, users will see an option to send files directly to a recipient's email address via the platform.
  • Enable parallel file uploading: Allows users to upload multiple files simultaneously rather than one by one.

    Warning: Enabling this significantly improves user speed but increases the instantaneous load on your server. Ensure your infrastructure can handle concurrent connections before enabling.

  • Chunk size limit: When uploading large files, the system breaks them into smaller "chunks" to prevent timeouts. Define the size (in MB) per chunk here.
    • Note: You cannot set this higher than your actual web server's (Nginx/Apache/PHP) upload_max_filesize.
  • Offload chunk size limit: Applicable only if you are using external storage (like AWS S3 or Wasabi). This controls the chunk size sent to the remote bucket.

File Types & Previews

  • Blacklisted extensions: A comma-separated list of file types users are forbidden to upload (e.g., exe, bat, sh). Use this to prevent malware distribution.
  • Preview extensions: A comma-separated list of file types that can be viewed directly in the browser without downloading (e.g., jpg, png, pdf, mp4).

System Modules (Feature Toggles)

Enable or disable major sections of the platform based on your business model.

  • Enable the Transfers report system: Adds a "Flag" icon to download pages, allowing visitors to report illegal or abusive content to admins.
  • Enable the Projects system: Allows users to create folders or workspaces to organize their files.
  • Enable the pixels system: Allows users to add tracking IDs (Facebook Pixel, Google Analytics) to their download pages for marketing data.

Custom Domains & DNS

These settings allow users to brand their links (e.g., files.mycompany.com instead of birbshare.com).

  • Enable the Custom Domains system: The master switch to allow users to attach their own domains.
  • Enable the Additional Custom Domains system: Allows you (the admin) to add global domains that all users can select from a dropdown list.
  • Enable the usage of the main domain: If disabled, users must use a custom domain or an additional domain; they cannot generate links using your primary site URL.
  • IP of your server: The specific IP address that users must point their DNS A-Records to when setting up a custom domain.

Security & Blacklists

  • Blacklisted domains: A list of domains that cannot be used as a destination or referrer.
  • Blacklisted keywords: A comma-separated list of words forbidden in custom URL aliases. Use this to reserve system paths (like admin, login, api) or prevent offensive words in URLs.
