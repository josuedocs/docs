BirbGames Admin Guide

Welcome to the BirbGames Admin Backend! This documentation serves as your comprehensive guide to navigating and managing the core functionality of the platform. The left-hand navigation panel is your control center, logically ordered to ensure you can quickly access every section.

Let's walk through each menu item in order, from top to bottom.

Dashboard

The Dashboard is your mission control. Think of this as the platform’s pulse monitor. It’s the first page you see, designed to give you an immediate, high-level overview of the most critical metrics, which are organized into four dynamic tabs:

USERS The USERS tab tracks the growth and activity of the user base. This includes visualizations of the User baseover time, the current and previous period Sign-ups, and a Sign-ups by source breakdown (e.g., Email, Social Media). This data is essential for assessing marketing performance and platform reach.

BETS The BETS tab provides a crucial analysis of betting volume and profitability. Key metrics include Gross Casino Revenue (GCR) and GCR margin comparisons, historical charts of Wagered amount and GCR, plus detailed tables breaking down performance by Original games, Top winners, and Top losers. This tab is your primary tool for monitoring gaming performance.

BONUSES The BONUSES tab focuses specifically on the cost and usage of promotional incentives. This view displays a timeline of Bonuses paid and a Detailed breakdown table showing the count, minimum, maximum, average, and total value of bonuses distributed by type. It helps track promotional expenditure and impact.

ACCOUNTING The ACCOUNTING tab presents the full financial picture, centering on Net Gaming Revenue (NGR), calculated as: NGR = GGR - Bonuses - Affiliate commissions - Credits + Debits + Raffle fees. It features dual metrics for Balance (Deposits vs. Withdrawals) and Deposits (previous vs. current period), alongside charts for Deposited amount and Withdrawn amount, and status breakdowns for both deposits and withdrawals. This is the financial summary for the platform.

Users

The Users section is the central hub for user management, providing a full, filterable list of every registered account. This is where you conduct account audits, security checks, and operational tasks.

The user list displays key initial information in columns:

  • ID: The unique numerical identifier for the user's account.
  • Name: The user's display name or username.
  • Email: The verified email address associated with the account.
  • Joined: The duration since the user registered on the platform.
  • Last seen: How recently the user was active on the platform.

Each user row includes a three-dot action menu offering granular control over the account:

  • View: Opens a comprehensive, read-only profile view for review, including history and KYC status.
  • Edit: Allows administrative modification of user details, such as name, status (e.g., suspended/active), and internal notes.
  • Delete: Permanently removes the user account and associated data (use with extreme caution).
  • E-mail: Initiates a direct message or action targeting the user's registered email address.
  • Devices: Displays a list of all devices (browser, OS, type) the user has logged in from, useful for security checks.
  • IP History: Shows a chronological list of IP addresses used to access the account, critical for fraud detection and ensuring account integrity.

Accounts

The Accounts section provides a financial view of every user's wallet, allowing for audit and manual adjustment of funds. This section focuses specifically on the transactional integrity of the platform.

The account list displays key financial information in columns:

  • ID: The unique numerical identifier for the user's account.
  • Name: The user's display name or username.
  • Balance: The current monetary balance in the account.
  • Currency: The primary currency the balance is held in.
  • Updated: The time elapsed since the last transaction or balance update.

Each account row includes a three-dot action menu for detailed financial control:

  • View: Opens the account's full financial ledger for audit.
  • Transactions: Displays a list of all deposits, withdrawals, and wagers associated with the account.
  • Summary: Provides a high-level overview of the account's financial activity (total deposits, total withdrawals, etc.).
  • Debit: Allows an administrator to manually subtract funds from the user's balance for adjustments or corrections (requires justification and caution).
  • Credit: Allows an administrator to manually add funds to the user's balance (e.g., for promotional awards or error correction).

Currencies

The Currencies section allows you to manage the specific fiat and cryptocurrencies available on the platform, including their exchange rates and display settings.

The currency list displays the following key details:

  • ID: The unique system identifier for the currency.
  • Name: The full display name of the currency (e.g., "Bitcoin", "Ethereum").
  • Code: The standard ticker symbol used throughout the platform (e.g., "BTC", "ETH").
  • External code: If applicable, the external identifier used by third-party services or wallets.
  • External rate: The configured exchange rate, likely relative to the platform's base currency.
  • Decimal places: The precision with which the currency amount is displayed and calculated.

The three-dot action menu on each row typically allows you to Edit these details (such as updating the exchange rate or name) or Delete a currency that is no longer supported.

Bets

The Bets section provides a comprehensive, auditable record of every single wager placed across all games on the platform. This is your primary tool for investigating game results, checking volatility, and ensuring fair play.

The bet log displays the following critical transaction details:

  • ID: The unique transaction identifier for the specific bet.
  • Name: The user who placed the bet.
  • Game: The title of the game played (e.g., Plinko, Blackjack, European Roulette).
  • Bet: The amount wagered by the user.
  • Win: The gross amount returned to the user (Bet + Profit).
  • Profit: The net gain (green) or net loss (red/negative) for the user on that specific bet.
  • Currency: The currency used for the wager (e.g., CREDIT).
  • Placed: The timestamp indicating when the bet was recorded.

Each bet transaction has a three-dot action menu, which has an option to View the full bet details.

Bonuses

This section manages promotional campaigns and user incentives. Here, you can:

  • Configure automated bonus rules, such as "Welcome Bonus" (e.g., 100% match on first deposit). You can track usage and associated revenue here.

VIP

The VIP section manages your loyalty and reward programs, which are structured around four distinct tabs to give you complete control over user progression, rewards, and status.

Groups (VIP Tiers)

This tab defines the high-level tiers of the VIP program, often represented by badges or colors (e.g., Bronze, Silver, Gold, Diamond).

The Rank Groups list displays:

  • ID: The unique system ID for the group.
  • Icon: The visual badge or icon associated with the group.
  • Name: The display name of the tier (e.g., Bronze).
  • Display order: The numerical sequence in which the tiers should be presented to users.

The action menu allows you to Edit the group, which opens a detailed view to configure the Name, Color (e.g., a hex code like #cd7f32), Icon assets, Display order, and a list of specific Benefits associated with achieving this group level (like a "Level up bonus").

Ranks (Progression Levels)

The Ranks tab defines the granular progression levels within each Group, typically linked to experience points (XP).

The Ranks list displays:

  • ID: The unique rank ID.
  • Name: The specific name of the rank (e.g., Bronze 1, Silver 3).
  • Group: The high-level group this rank belongs to.
  • Min XP: The minimum Experience Points a user needs to reach this rank.
  • Bonus amount / Bonus currency: The specific monetary reward awarded upon achieving this rank.

This section is where you manage the entire XP progression ladder and ensure the rewards are correctly configured for each step.

Rates (XP Earning Logic)

The Rates tab manages the formula for how users earn the Experience Points (XP) required to advance through the Ranks.

This shows the Type of activity (e.g., "Bet"), the Currency involved (e.g., "CREDIT"), and the Rate which defines how much XP is earned per unit of currency spent (e.g., "1 CREDIT = 1.00 XP"). You can edit this rate to adjust the overall pace of progression across the entire platform.

Transactions (XP History)

The Transactions tab provides a full, auditable log of every Experience Point transfer that has occurred on the platform.

The transaction log displays:

  • ID: The unique transaction ID.
  • User: The user whose XP balance was affected.
  • Source: The reason for the XP change (e.g., bet placement, rank achievement, manual admin adjustment).
  • XP: The amount of XP added or subtracted.
  • Created at: The timestamp of the transaction.

This log is essential for auditing loyalty program integrity and investigating user progression issues.

Promo codes

The Promo codes section is where you manage static promotional campaigns designed to reward users with cash, bonuses, or other incentives. This tool is vital for marketing initiatives and direct player engagement.

The main management list displays:

  • ID: The unique system identifier.
  • Code: The actual, redeemable text string (e.g., SUMMER25).
  • Title: A descriptive name for the internal management of the code.
  • Type: The kind of reward (e.g., "Cash", "Bonus").
  • Enabled: A boolean indicator showing if the code is currently active and redeemable.
  • Redemptions: The total count of times the code has been successfully used by players.
  • Created: The timestamp indicating when the code was created.

When adding or editing a code, the Create promo code interface requires you to define critical parameters, including:

  • Code / Title: The public-facing code and internal title.
  • Type: The reward category (e.g., Cash, Bonus).
  • Enabled: Toggle to immediately activate or deactivate the code.
  • Min wager amount / Min wager day count: Requirements users must meet before redeeming the code.
  • Max redemptions: The total number of times the code can be used platform-wide.
  • Amount: The value of the reward given upon successful redemption.

Deposits

The Deposits section provides a real-time, auditable ledger of all incoming funds, whether they are completed, canceled, or pending. This log is crucial for reconciling customer payments and managing the funding process.

The deposit log displays detailed transaction information:

  • ID: The unique system identifier for the deposit record.
  • Name: The user who initiated the deposit.
  • Method: The payment gateway or method used (e.g., BIRB, Crypto Wallet).
  • Payment amount / Payment currency: The amount and currency sent by the user via the external payment method (e.g., 0.1 BIRB).
  • Account amount / Account currency: The amount and currency credited to the user's internal platform account after any exchange or processing (e.g., 100 CREDIT).
  • Status: The current state of the deposit (e.g., Completed or Cancelled).
  • Created: The timestamp of when the deposit was initiated.

Each deposit record has a three-dot action menu, offering View and Transaction (for sucessful transactions) options to allow administrators to examine the full payment information including external confirmations.

Withdrawals

The Withdrawals section provides the transaction log for all outgoing payments, managing the queue of requests from users to cash out their funds. This is a high-security and critical area for financial compliance and fraud prevention.

The withdrawal log displays detailed transaction information, which mirrors the deposit process but in reverse:

  • ID: The unique system identifier for the withdrawal record.
  • Name: The user who initiated the withdrawal.
  • Method: The payment gateway or method used for the payout (e.g., BIRB, Crypto Wallet).
  • Payment amount / Payment currency: The amount and currency sent to the user via the external payment method (e.g., 0.001 BIRB).
  • Account amount / Account currency: The amount and currency debited from the user's internal platform account (e.g., 1 CREDIT).
  • Status: The current state of the withdrawal request (e.g., Completed, Pending, or Rejected).
  • Created: The timestamp of when the withdrawal was initiated.

Each withdrawal record has a three-dot action menu, typically offering View and Transaction options to allow administrators to examine the full payment lifecycle, including internal debiting and external payout confirmation. For pending requests, this menu may also include actions like Approve or Reject.

Deposit methods

The Deposit methods section is where administrators manage the available options for users to fund their accounts and can add new payment options. This list shows all configured payment gateways, but importantly, BIRB is the currently Enabled and active payment method used by BirbGames. Other methods are listed but currently deactivated. The administrator can manage each method's status, gateway, and specific configuration options here.

Withdrawal methods

The Withdrawal methods section is where administrators manage the available options for users to cash out their funds. Crucially, administrators can add new withdrawal methods here as needed. This list shows all configured withdrawal gateways and systems. BIRB is the currently Enabled and active withdrawal method used by BirbGames, ensuring quick and seamless payouts. The administrator can manage each method's status, gateway, and specific configuration options here.

Chat

This section is for moderating and managing the platform's public or internal chat feature. Tools here may include:

  • Reviewing chat logs for compliance.
  • Banning or muting users in the chat room.
  • Setting up automated moderation rules (e.g., filtering profanity).

Settings

The main configuration hub for the entire platform. This usually contains global settings that affect the user experience, such as:

  • Site metadata (title, footer text).
  • Timezone and locale defaults.
  • Security settings and API key management.

Maintenance

This area is designed for operational tasks, especially when preparing for or recovering from updates. Common features include:

  • Switching the site to "Maintenance Mode" (offline for users).
  • Clearing system caches.
  • Running system health checks or data cleanup scripts.

Add-ons

If your platform uses third-party extensions, custom modules, or optional features, the Add-ons section is where they are installed, configured, and managed.

Help

This section is your internal knowledge base. It typically contains links to internal team guides, support contact information, or direct access to platform-specific developer documentation.

Original games

This dedicated section provides specific management tools for the proprietary or "in-house" games developed for the Birb platform. Configuration options are typically unique to the game logic, such as:

  • Setting house edge parameters.
  • Managing game versions and deployment.
  • Monitoring game-specific analytics.

English (Language Selector)

This option allows you to change the language and localization of the Admin Panel Interface itself, ensuring the backend team can work in their preferred language.
