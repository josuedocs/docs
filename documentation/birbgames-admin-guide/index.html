BirbGames Admin Guide

Welcome to the BirbGames Admin Backend! This documentation serves as your comprehensive guide to navigating and managing the core functionality of the platform. The left-hand navigation panel is your control center, logically ordered to ensure you can quickly access every section.

Let's walk through each menu item in order, from top to bottom.

Dashboard

The Dashboard is your mission control. Think of this as the platform’s pulse monitor. It’s the first page you see, designed to give you an immediate, high-level overview of the most critical metrics, such as:

  • Real-time statistics (e.g., number of active users, total transactions).
  • Recent activity logs and system alerts.
  • Key performance indicators (KPIs) like total revenue or new registrations over a period.

Users

The Users section is where you manage individual user profiles. This is the central hub for managing the people on the platform, allowing you to:

  • Searching and filtering all registered users.
  • Viewing personal details, registration dates, and security logs.
  • Managing Know Your Customer (KYC) documentation.
  • Enabling, disabling, or banning user accounts.

Accounts

While Users focuses on the person, Accounts focuses on their financial ledger. Here, you can review and adjust:

  • The current balance of a user’s funds across different currencies.
  • Detailed transaction histories (separate from specific deposit/withdrawal logs).
  • Internal account adjustments or corrections.

Currencies

This is where you define and manage all accepted fiat and cryptocurrency types on the platform. This includes:

  • Activating or deactivating specific currency support.
  • Setting up internal conversion rates (if applicable).
  • Managing currency display formats and decimal places.

Bets

The Bets section provides a comprehensive, auditable record of every single wager placed on the platform. You can usually filter this data to review:

  • Winning and losing bets.
  • Bets placed by specific users or on specific games.
  • Details like timestamps, amounts, and associated profit/loss.

Bonuses

This section manages promotional campaigns and user incentives. Here, you can:

  • Configure automated bonus rules, such as "Welcome Bonus" (e.g., 100% match on first deposit). You can track usage and associated revenue here.

VIP

The VIP section manages your loyalty and reward programs. Use this area to:

  • Define and configure the tiers of your VIP program (e.g., Bronze, Silver, Gold).
  • Manually assign or change a user's VIP status.
  • Configure the benefits and perks associated with each level.

Promo codes

This section manages promotional campaigns and user incentives. Here, you can:

  • Create, activate, and deactivate static promotional codes (e.g., SUMMER25) used for marketing campaigns, including setting usage limits and expiry dates.

Deposits

This section manages the processing queue for incoming fund transfers. It provides:

  • A log of all incoming funds, often requiring manual review or confirmation, especially for certain payment methods.

Withdrawals

This section manages the processing queue for outgoing fund transfers.

  • This is the queue for user requests to cash out. This is a high-security area where requests are typically reviewed, approved, or rejected based on security protocols and balance checks.

Deposit methods

This section handles the technical setup for how money moves into the platform:

  • Configure and enable payment gateways (e.g., credit card processors, crypto wallet connections, bank transfers) for funding user accounts.

Withdrawal methods

This section handles the technical setup for how money moves out of the platform:

  • Define the options users have for receiving funds, linking them to specific processors or banking systems.

Chat

This section is for moderating and managing the platform's public or internal chat feature. Tools here may include:

  • Reviewing chat logs for compliance.
  • Banning or muting users in the chat room.
  • Setting up automated moderation rules (e.g., filtering profanity).

Settings

The main configuration hub for the entire platform. This usually contains global settings that affect the user experience, such as:

  • Site metadata (title, footer text).
  • Timezone and locale defaults.
  • Security settings and API key management.

Maintenance

This area is designed for operational tasks, especially when preparing for or recovering from updates. Common features include:

  • Switching the site to "Maintenance Mode" (offline for users).
  • Clearing system caches.
  • Running system health checks or data cleanup scripts.

Add-ons

If your platform uses third-party extensions, custom modules, or optional features, the Add-ons section is where they are installed, configured, and managed.

Help

This section is your internal knowledge base. It typically contains links to internal team guides, support contact information, or direct access to platform-specific developer documentation.

Original games

This dedicated section provides specific management tools for the proprietary or "in-house" games developed for the Birb platform. Configuration options are typically unique to the game logic, such as:

  • Setting house edge parameters.
  • Managing game versions and deployment.
  • Monitoring game-specific analytics.

English (Language Selector)

This option allows you to change the language and localization of the Admin Panel Interface itself, ensuring the backend team can work in their preferred language.
